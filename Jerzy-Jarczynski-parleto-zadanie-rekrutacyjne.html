Improvements

Removing W3C errors and warnings

  1. Adding lang attribute to html

    The 'lang' attribute helps to improve accessibility. Search engines use the 'lang' attribute to better understand the language of the content. 'lang' attribute is crucial for indicating the language of each page. Specifying the language can enhance the overall user experience. The 'lang' attribute can also be used in conjuction with CSS for language-specific styling.

  2. CSS instead SCSS in style tags

    CSS is simpler and more straightforward than SCSS. Since SCSS needs to be compiled into CSS before it's interpreted by the browser, using plain CSS in style tags can eliminate the compilation step, potentially leading to faster rendering times. Using plain CSS ensures broader compatibility without relying on a preprocessor. For small projects or quick edits, working directly with CSS in style tags can be more convenient.

  3. no legacy JavaScript

    Avoiding legacy JavaScript reduces maintenance overhead, improves performance, and ensures better compatibility with modern web standards.

  4. fixing typoos and html bugs

    Fixing typos and HTML bugs enhances code accuracy and improves website functionality.

  5. fixing error in sizes attribute in img

    Finally I've decided to resign from developing image with set of sources.

  6. missing src attribute in img

    The src attribute in img tags is crucial for displaying images; its absence results in a broken or missing image on the webpage.

  7. missing alt attribute in img

    Including the alt attribute in img tags is essential for accessibility, providing text descriptions for images, aiding users with disabilities and enhancing SEO.

  8. duplicated id in input

    Avoiding duplicated IDs in input elements is vital for maintaining a valid HTML structure. Duplicate IDs can lead to unpredictable behavior in scripts and hinder accessibility features.

  9. illegal character in img src

    Using illegal characters in the src attribute of an img tag can break the image link and prevent proper loading. Ensure the src adheres to valid URL and file naming conventions.

  10. fixing nesting errors with div and hr inside headers

    Correcting nesting errors with div and hr inside headers ensures proper HTML structure. Place them outside the header or use appropriate container elements.

Fixing other bugs

  1. headers hierarchy - only 1 h1 on the page

    Maintain a proper headers hierarchy with only one h1 on the page. Use h2, h3, etc., for subheadings to create a logical and semantically correct structure.

  2. Add Responsive Web Design (RWD):

    Unfortunately, I ran out of time to implement Responsive Web Design for the website. It currently works well on screens up to 1400px wide. I regret this oversight, especially since RWD poses no challenge for me, whether in a mobile-first or desktop-first approach.   3. remove !important styles

    Removing !important styles enhances code maintainability and flexibility by allowing styles to be overridden more easily. It promotes a clearer and more predictable cascade in the stylesheet.

  4. unification of units

    Due to a lack of time for refactoring, there might be occasional pixels in the styles. However, overall, I attempted to adhere to the convention of using rem where possible. Additional benefits of unification of units include:

    • Consistency: Using a consistent unit like rem throughout the styles promotes a uniform and predictable appearance across the website.
    • Scalability: rem is relative to the root font size, making it easier to scale elements proportionally by adjusting the root font size.
    • Accessibility: Uniform units contribute to a more accessible design, accommodating users who may rely on text resizing or have different default font sizes set in their browsers.
    • Maintenance: Unifying units simplifies code maintenance by reducing the need for unit conversions and making stylesheets more straightforward to understand.
  5. replace unnecessary grid with horizontal flex

    Replacing unnecessary grid with horizontal flex offers simplicity, reduces code complexity, and enhances performance. It's advantageous for straightforward, one-dimensional layouts, providing a more concise and easily maintainable solution.

Fireworks

  1. sticky navbar

    Applying a sticky navbar ensures that the navigation is always visible on the page. In this case, the navigation bar facilitates swift navigation between sections, which can be very helpful. I didn't add an arrow directing to the top of the page; the logo serves this function instead.

  2. animated scroll to target section

    Animated transitions between sections positively impact UX. Navigation is not abrupt and sudden but smooth and calm, striking a balance — swift enough for efficient access to the desired section, yet not overly rapid.

  3. parallax backgrounds

    A parallax background is a simple and pleasant effect that adds space and breathes life into the webpage. The photograph used on the site is sourced from a free graphics bank and depicts a cityscape at night. To the text on the parallax, I added a text-shadow to enhance readability.

  4. form validation

    Form validation is fundamental in today's web development. Its absence in more significant projects is impractical and risky. An additional security measure is a captcha, which I didn't implement. The form works, but it currently only sends data to the browser console using console.log.

  5. modal on form submit

    A modal appearing after form submission positively enhances UX. Due to time constraints, I didn't focus on its appearance, but the modal opens and closes without errors in the console, informing the user that their message has been sent.

  6. accordion

    The accordion is sourced from the W3Schools base, with minimal modifications tailored to the color palette and overall style of the page. I deemed it a good way to present a small number of job offers in an engaging manner for the user.

Aesthetics

  1. color pallete

    I opted for a minimalist 4-color palette, utilizing it in a versatile manner. I created gradients and ensured suitable contrasts for readability, emphasizing important information. I added additional colors and their variations to CSS variables as needed.

  2. coherent typography

    I aimed to unify the typography by selecting two primary fonts used throughout the entire website. I initially defined colors and sizes globally, adjusting them as needed for specific sections. This approach ensures consistency and harmony in the text. The choice of fonts may not be the most deliberate; I selected Lobster for its distinctiveness, using it to emphasize the logo and strong elements on the page.

  3. font awesome icons

    I really enjoy using Font Awesome. The icons have a pleasing aesthetic, and it's easy to apply suitable styles to them. The extensive icon library makes it straightforward to find something appropriate for the specific purpose of the presentation.
